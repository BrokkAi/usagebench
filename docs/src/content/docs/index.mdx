<p class="snapshot-kicker">BIFROST CONFORMANCE · REVIEWED LSP-PARITY CORPUS</p>

<h2 class="landing-headline">Hold usage analysis to the standard developers already trust</h2>

UsageBench is Bifrost's recurring conformance and regression suite for source
usages and navigation. Its cases are derived from mature language-server tests,
existing analyzer suites, and difficult real-world edges. Popular LSPs provide
the baseline and post-review calibration: Bifrost should reach parity wherever
the LSP behavior matches the reviewed language semantics, and may deliberately
be more precise or cover additional static edges when the evidence supports it.

All 158 current cases have since completed a
[first human ground-truth audit](ground-truth-review/). The corpus now preserves
those decisions as executable regression contracts, and the daily workflow runs
them against Bifrost `master` so a previously satisfied semantic edge cannot
quietly disappear.

The report shape remains analyzer-neutral. Language servers are the first
comparison set, not a permanent limit: future analyzers can be added through the
same source-location contract without changing the authored ground truth.

These are contract results, not a general analyzer leaderboard. A policy-only
near miss contains every required location but also includes an import,
re-export, or export-metadata location that UsageBench excludes. A hard
disagreement can combine missing usages, broader symbol families, or a different
navigation target.

All 136 exact Bifrost outcomes are expected passes in the current baseline. The
five expected gaps are the cases that remain non-exact in the current run.
